func 1 = --何らかの処理
func 2 = --何らかの処理

課題4.11と4.12？

P81~P83

再帰は大事。例えば階乗。漸化式を作るイメージ。例えば階乗なる演算子を関数で書くと $$ !:R→R \quad !(n)=(n-1)!(n) $$ かつ $$ !(0)=1 $$ これをhaskellで書くと

fac n = fac (n-1)*n
fac 0 = 1

↑ガード記法でもかける。(P82)なお、上のやつは-1とかを入れると永遠に続いてしまう。

数学なら、例えば!(-1)は定義されていないという立場をとれるがプログラミングの場合はそうもいかない。そこでotherwiseを使う。

課題P84(4.17と4.18)

P84~P85

P85の上にpower関数がある。$2^n$乗を求める再帰関数。

sumFacs n = fac 0 + fac 1 + … + fac (n-1) + fac n

これは数学で書くと $$ \sum_{k=1}^n k $$ よって

sumFacs n
    | n == 0 = 1
    | n > 0 = sumFacs (n-1) + n

同様に $$ \sum_{k=1}^n f(k) $$

sumFun f n
    | n == 0 = f 0
    | n > 0 = sumFacs (n-1) + f(n)

P97

タプル()とリスト[]

  • タプルは違う型を数の制限を設けて定義。
  • リストは同じ型を数の制限無しで定義。
("Salt: 1kg",139)

などといった感じで違うデータ型を一つのタプル内に混在できる。型でいうと

(String,Int)

さらにエイリアス(別名)も定義できる。

type ShopItem = (String,Int)

ジャンケンの例でみたようにdataは新しい型を定義する。

リストの場合は同一型縛りがるため、型宣言は要素がなんこあろと

[(String,Int)]
[ShopItem]

などとできる。
